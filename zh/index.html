技能生命周期（RSP）
  1. 注册 —— 发布意图、约束、上下文与可观测结果
  2. 发现 —— 按目标与环境检索技能
  3. 匹配与排序 —— Agent 在策略/约束下选择最优技能
  4. 执行 —— 运行时绑定参数并强制约束
  5. 反馈 —— 记录结果，提升选择与鲁棒性

为什么需要 Robot Skill Store

当今的机器人能力高度碎片化：技能往往深埋在代码里，与特定机器人/环境强耦合，为单一场景定制，难以在团队或系统之间复用。

我们口中的“技能”，很多时候只是一个函数调用、一个 ROS 节点，或一个只能跑一次的 demo 脚本。

这种碎片化直接导致：

  • 能力难以跨机器人、跨团队共享
  • 安全与约束缺少结构化表达，难以系统性推理
  • 仿真到真实部署难以规模化迁移
  • 行为编排依赖硬编码拼接，Agent 难以组合与决策

Robot Skill Store 正是为解决这一根本性问题而设计。

什么是机器人技能

在 Robot Skill Store 中，技能不仅仅是可执行代码。一个机器人技能，是最小的、有意义且安全的能力单元，其定义远不止于“怎么实现”。

一个技能明确包含
  • 意图（Intent）——机器人要达成什么目标
  • 约束（Constraints）——安全边界、限制与前提假设
  • 执行上下文（Context）——仿真或真实世界运行条件
  • 可观测结果（Outcomes）——成功/失败标准与反馈信号
因此技能可以被
  • 发现与比较
  • 执行前验证
  • 由 Agent 组合与决策
  • 跨机器人/环境复用

技能不再是私有实现细节，而是由协议定义的共享能力对象。

RSP：Robot Skill Protocol

要让技能可移植、可比较且安全，必须被协议化。Robot Skill Protocol（RSP）定义：

  • 技能如何注册与描述
  • Agent 如何发现、匹配与选择技能
  • 执行过程中如何约束与监控
  • 结果如何记录并反馈复用

RSP 不是 SDK，也不是框架；它是技能提供方、Agent/编排系统、仿真器与真实机器人之间的共同语言。

Demo ≠ Demo

Robot Skill Store 的 demo 不是展示效果，而是参考执行模型（Reference Execution Model）：强调可复现的绑定、约束强制与可审计结果，在仿真与真实之间保持一致。

参考模型聚焦
  • 显式技能规格
  • 运行时参数覆盖（Base/TCP/环境）
  • 环境感知的执行行为
  • 仿真↔真实的一致性评估

面向新一代机器人生态

Robot Skill Store 面向正在构建未来机器人的团队：中小规模机器人研发团队、工业自动化开发者、具身智能/Agent 研究者，以及平台与生态构建者。

如果你相信机器人智能应当模块化而非单体、由Agent 驱动而非硬编码、安全应当内生于能力定义——Robot Skill Store 正是为你而生。
