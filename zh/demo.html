UR5 参考执行模型

Robot Skill Store 的 demo 不是展示效果，而是参考执行模型：强调正确性、可复现性与可扩展性，展示基于 RSP 的技能如何在仿真与真实之间一致执行。

参考执行模型（泳道式流程 SVG）
强调：显式规格 + 运行时绑定 + 约束强制 + 可审计结果（仿真/真实一致）。

UR5 示例展示了什么

  • 技能如何被规格化定义（意图/约束/上下文/结果）
  • 执行上下文如何绑定（Base/TCP/环境）
  • 约束如何在运行时被强制
  • 结果如何被记录、评估并反馈复用

运行时参数绑定

执行时，运行时会把技能规格绑定到具体系统：机器人基座位姿、TCP 定义、环境参数等。参数变化不应改变技能语义，而应由运行时做一致性处理。

执行流程

  1. Agent 发现并选择技能
  2. 运行时验证上下文与约束（执行前）
  3. 绑定 Base/TCP/环境参数
  4. 在 UR5（或仿真）上执行并监控
  5. 记录可审计结果与反馈，回传系统

仿真与现实的一致性

同一技能规范贯穿仿真、测试与真实部署。通过一致的绑定、约束强制与结果采集，降低部署风险，加速迭代。

这个 Demo 是什么 / 不是什么

这是
  • 规格优先：由 Skill Spec + Binding 驱动
  • 快速失败：约束违规会输出证据
  • 可审计：每次运行都有证据包
  • 可比较：支持回归对比
不是
  • ROS / URScript 教程
  • 控制器实现细节
  • 硬件 bring-up 指南
  • 特定应用流程方案
