智能体

智能体决定“要尝试什么”；RSS 平台决定“允许真正发生什么”。

在 Robot Skill Store（RSS）里，智能体是用于选择、组合与调度技能的决策层。它不能绕过治理、评测或执行真相——任何动作最终都要变成平台管理的 Run，并绑定不可变证据。

一条不可协商的规则

智能体可以决定尝试什么；RSS 强制执行什么能真正发生。

  • 智能体提出候选动作：选择哪个 Skill/版本、使用哪个 Profile、带哪些输入。
  • RSS 将动作固化为 Run，执行 Policy 门禁，并记录 Evidence。
  • “成功”由 Benchmark 与结果判定，而不是由智能体“说服”。

什么是智能体（在 RSS 语境下）

智能体是一个编排器，它基于目标、上下文与历史证据来选择哪个技能版本去运行，并决定采用哪个 Profile 与参数。

智能体可以 / 不可以做什么

可以

  • 在策略允许范围内选择 Skill 与 Skill Version
  • 绑定 Profile 与参数（机器人/环境/限制）
  • 组合多步流程（workflow）
  • 根据 Run 结果分支 / 重试 / 升级
  • 用 benchmark 证据比较候选方案

不可以

  • 直接执行代码（不允许“隐藏路径”）
  • 绕过策略门禁（安全/许可/审批）
  • 自行判定“成功”（由 benchmark 判定）
  • 修改记录（Run 不可变）
  • 随意改语义（契约与字段是版本化的）

智能体在系统中的位置

  1. 智能体：生成候选计划与动作意图（要跑什么）。
  2. RSS 平台：把动作转换为 Run（谁/何时/如何跑）。
  3. RSP 不变量：强制对象与门禁（允许什么发生）。
  4. Evidence：记录日志/指标/轨迹/视频，支撑审计与复现。

为什么要把智能体“关进平台”

  • 可治理：审批/授权/风控要可追溯。
  • 可评测：跨版本、跨团队比较必须有统一证据。
  • 可复现：同一输入同一版本应产出可重放的 Run 记录。
  • 可扩展：规模化生产不能依赖 UI 操作或个人经验。
