Robot Skill vs LLM Skill

理解两者的关系与差异

为什么机器人技能与 LLM 技能处于不同层次、如何互补，以及 RSS 在 AI–机器人技术栈中的定位。

Q1: RSS 中的 Robot Skill 是什么？

RSS 中的 Robot Skill 是一种可确定执行的机器人能力单元，在明确约束下控制真实机器人，并能够被验证、认证与复用。

  • 可在真实机器人上执行
  • 确定性行为
  • 明确的前置条件与约束
  • 可通过证据进行验证

Q2: Robot Skill 等同于 LLM Skill 吗？

不是。 LLM Skill 主要作用于认知与工具调用层，而 RSS 的 Robot Skill 直接作用于物理执行层。它们解决的是完全不同的问题空间。

Q4: 两者有关联吗？

有关联。LLM Skill 负责决策与编排，Robot Skill 负责安全、稳定的物理执行。

🤝

Q5: 能集成吗？

可以。RSS 的 Robot Skill 可作为 Agent 系统调用的物理执行基元。

⚠️

Q6: 为何不直接控制？

LLM 直接控制机器人适合探索，但在生产环境风险过高。

📍

Q7: RSS 在什么位置？

RSS 是智能决策系统与物理机器人之间的可信物理执行层。

💎

Q8: 为什么强调技能资产？

机器人技能研发成本高、验证难，但一旦证明可靠就极具价值。

RSS 理念

"RSS 将机器人技能视为可认证、可复用的基础设施级能力资产，而非一次性 Demo。"
