如何提交技能

从你的机器人能力到平台认证技能，共四步。

1

定义技能规格（Skill Spec）

JSON 文档：声明意图、接口（输入/输出）、前/后置条件、安全约束与错误模式。规格即合约，后续流程均以此为基础。

2

在平台注册技能

POST /skills — RSS 校验 Schema，分配 skill_id，状态设为 REGISTERED，技能立即在目录中可被检索。

3

提交验证执行记录

每次执行调用 POST /runs（结果、约束、原因）。制品通过 POST /runs/{run_id}/artifacts 挂载，不可篡改存储。

4

认证与治理

GET /skills/{skill_id}/{version}/governance 计算快照——成功率、违规率、回归。通过后状态晋升为 CERTIFIED，技能公开发布。

RSS 验证什么

  • 规格合规性——接口、约束与错误模式在注册时 Schema 校验。
  • 执行结果——每次运行记录结果、违约约束与原因。
  • 制品完整性——日志、视频、遥测与对应 Run 绑定，上传后不可修改。
  • 滚动成功率——在窗口（如最近 30 次）上评估，而非单次结果。
  • 违规率——安全硬限制触发与软失败分别追踪。
  • 回归检测——已认证技能成功率骤降时自动触发 DEGRADED。
证据等级：随 Run 数增加，解锁更广泛的部署许可与更高信任信号。

RSS-kit：开发者工具包

RSS-kit 是一个 Python 包，将平台 API 封装为 CLI 和代码生成器——让你专注于技能逻辑，而非 HTTP 调用细节。它可以脚手架规格文件、本地校验，并一条命令提交运行记录。

rss-kit · 三步工作流
1
安装并脚手架新技能
pip install rss-kit rss-kit new peg_in_hole --template assembly --robot UR10e
2
在提交前本地校验规格
rss-kit validate peg_in_hole_manifest.json
3
向平台提交执行记录
rss-kit submit --skill peg_in_hole --version 1.0.0 \ --outcome SUCCESS --env real_robot \ --platform-url http://localhost:8080

RSS-kit 还可从技能规格脚手架 ROS 2 包，并支持从平台拉取已有规格用于本地开发。  RSS-kit 文档

技能规格格式

最小可行结构示例——完整参考见文档。

技能规格 — 最小结构（JSON）
{
  "skill_id": "peg_in_hole",
  "version": "1.0.0",
  "name": "孔轴插入",
  "category": "Assembly",
  "maturity": "beta",
  "intent": "使用柔顺运动将销轴以紧公差插入孔中。",
  "interface": {
    "inputs": {
      "hole_pose":   { "type": "pose",  "frame": "world" },
      "peg_length":  { "type": "float", "unit": "mm" },
      "max_force_n": { "type": "float", "unit": "N", "robot_relative": true }
    },
    "outputs": {
      "outcome":         { "type": "string", "enum": ["SUCCESS", "FAILURE"] },
      "insertion_depth": { "type": "float",  "unit": "mm" }
    }
  },
  "conditions": {
    "pre":  [{ "id": "peg_grasped",    "check": "gripper.closed == true" }],
    "post": [{ "id": "fully_inserted", "check": "insertion_depth >= peg_length * 0.95" }]
  },
  "error_schema": {
    "FORCE_EXCEEDED":   { "severity": "CRITICAL", "recoverable": false },
    "SEARCH_EXHAUSTED": { "severity": "HIGH",     "recoverable": true  }
  },
  "deployment": {
    "requires_capabilities": {
      "min_dof": 6,
      "requires_force_torque_sensor": true
    }
  }
}

机器人兼容性

技能声明能力需求（自由度、负载、力矩传感器、末端执行器类型）与逐机型参数配置，平台运行时完成匹配。

UR5e, UR10e, UR16e, UR20 · KUKA iiwa7, KR10, KR20, KR200 · ABB IRB1200, IRB2400 · Fanuc LRMate200iD, M10iA, M20iA, R2000iB, ARC Mate 100iD, P250

新机型？在技能规格中添加 robot_profiles 字段，声明该机型的参数边界即可。
