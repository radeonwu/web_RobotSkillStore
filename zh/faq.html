Robot Skill vs LLM Skill（8 问答）

Q1. RSS 中的 Robot Skill 是什么？

RSS 中的 Robot Skill 是一种可确定执行的机器人能力单元，在明确约束下控制真实机器人，并能够被验证、认证与复用。

  • 可在真实机器人上执行
  • 确定性行为
  • 明确的前置条件与约束
  • 可通过证据进行验证

Q2. Robot Skill 等同于 LLM Skill 吗？

不是。 LLM Skill 主要作用于认知与工具调用层，而 RSS 的 Robot Skill 直接作用于物理世界中的机器人执行层。

Q3. 关键差异是什么？

维度LLM SkillRSS Robot Skill
执行域语言 / 工具 / API物理机器人
确定性概率性确定性
失败代价低高（安全、硬件、产线）
评估方式语义 / 奖励驱动证据驱动的物理指标
复用门槛低高（核心护城河）

解释：执行代价与风险越高，验证与治理越关键。

Q4. 两者有关联吗？

有关联，但处在不同层级。

LLM Skill 负责“决策与编排”，Robot Skill 负责“安全、稳定的物理执行”。

Q5. RSS 能与 Agent/LLM 系统集成吗？

可以。RSS 的 Robot Skill 可作为 Agent 系统调用的物理执行基元，但执行必须保持可观测、可验证与可治理 —— 这正是 RSS 提供的能力。

Q6. 为什么不让 LLM 直接控制机器人？

LLM 直接控制机器人适合探索，但在生产环境风险过高。RSS 的存在，是为了在真实世界中提供确定性与工程级保障。

Q7. RSS 在 AI/机器人技术栈中处于什么位置？

RSS 是智能决策系统与物理机器人之间的技能验证、认证与交付层。它不替代 Agent 或模型，而是让它们“落地到现实”。

RSS 不是替代大模型或 Agent，而是为它们提供可信、可落地的物理执行层。

Q8. 为什么强调 Robot Skill 是资产？

机器人技能研发成本高、验证难，但一旦证明可靠就极具价值。RSS 将技能从一次性项目代码变成可长期复用、可复利积累的能力资产。

RSS 将机器人技能视为可认证、可复用的基础设施级能力资产，而非一次性 Demo。
