Robot Skill vs LLM Skill（RSS 官方 FAQ）

LLM Skill 负责“决策与编排”，Robot Skill 负责“安全、稳定的物理执行”。

Q1. RSS 中的 Robot Skill 是什么？

RSS 中的 Robot Skill 是一种可确定执行的机器人能力单元，在明确约束下控制真实机器人，并能够被验证、认证与复用。

Q2. Robot Skill 等同于 LLM Skill 吗？

不等同。 LLM Skill 主要作用于认知与工具调用层，而 RSS 的 Robot Skill 直接作用于物理世界中的机器人执行层。

Q3. 关键差异是什么？

维度LLM SkillRSS Robot Skill
执行域语言 / 工具 / API物理机器人

失败成本低高（安全/硬件/生产）

解释：执行成本与风险越高，验证与治理越关键。

Q4. 两者有什么关系？

相关但分层：LLM Skill 决定“做什么”，Robot Skill 保证“怎么做才安全可靠”。

Q5. RSS 能与 Agent/LLM 系统集成吗？

可以。RSS 的 Robot Skill 可作为 Agent 系统调用的物理执行基元，但执行过程必须保持可观测、可验证与可治理。

Q6. 为什么不让 LLM 直接控制机器人？

探索阶段很有价值，但生产环境风险过高：物理执行有真实后果，安全、可重复与认证都很重要，工业部署需要确定性保障。

Q7. RSS 在 AI/机器人技术栈中定位在哪里？

RSS 是连接智能决策系统与物理机器人之间的“技能验证 / 认证 / 交付层”。它不是替代模型或 Agent，而是让它们“落地到真实世界”。

RSS 不是替代大模型或 Agent，而是为它们提供可信、可落地的物理执行层。

Q8. 为什么强调把 Robot Skill 当作资产？

机器人技能构建成本高、验证困难，一旦证明可靠就具备长期价值。RSS 将技能从一次性项目代码，转成可长期复用、可积累的能力资产。

RSS 将机器人技能视为可认证、可复用的基础设施级能力资产，而非一次性 Demo。
