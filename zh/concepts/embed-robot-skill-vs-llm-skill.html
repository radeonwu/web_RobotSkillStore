机器人技能 vs LLM 技能（RSS 官方 FAQ）

Q1. RSS 中的机器人技能是什么？

RSS 中的机器人技能是一种确定性的、可执行的能力，在显式约束下控制物理机器人。它被设计为可验证、可认证，并可在不同部署场景中复用。

  • 可在真实机器人上执行
  • 确定性行为
  • 显式前置条件与约束
  • 可通过证据验证

Q2. 机器人技能与 LLM 技能是同一回事吗？

不是。 LLM 技能工作于认知或工具调用层，而 RSS 机器人技能工作于物理执行层。两者解决的是本质不同的问题空间。

Q3. 两者的关键区别是什么？

维度LLM 技能RSS 机器人技能
执行域语言 / 工具 / API物理机器人
确定性概率性确定性
失败代价低高（安全、硬件、生产）
评估方式语义 / 基于奖励基于证据的物理指标
复用壁垒低高（核心护城河）

解读：执行成本与风险越高，验证与治理就越关键。

Q4. 机器人技能与 LLM 技能有关联吗？

有，但它们工作于不同的层次。

LLM 技能决定做什么。 机器人技能保障如何安全可靠地执行。

Q5. RSS 能与基于 Agent 或 LLM 的系统集成吗？

可以。RSS 机器人技能可作为基于 Agent 系统的物理执行原语。Agent 可编排技能，但执行必须保持确定性、可观测性和可认证性——这正是 RSS 所提供的。

Q6. 为什么不让 LLM 直接控制机器人？

LLM 直接控制机器人在探索阶段可能很强大，但用于生产存在风险。物理执行有真实世界的后果；安全性、可重复性和认证至关重要；工业部署需要确定性保证。

Q7. RSS 在 AI / 机器人技术栈中的定位是什么？

RSS 是智能决策系统与物理机器人之间的技能验证、认证与交付层。它不取代 Agent 或模型——而是将它们锚定于现实。

Q8. 为什么将机器人技能视为资产？

机器人技能构建成本高、验证困难，但一旦经过验证则极具价值。RSS 将机器人技能从项目特定代码转变为可长期积累的能力资产。

RSS 将机器人技能视为可认证、可复用的基础设施资产，而非演示程序。
