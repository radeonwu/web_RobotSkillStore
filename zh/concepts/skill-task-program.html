Skill ≠ Task ≠ Program

RSS 强调把“可复用能力”从一次性流程与代码里抽离出来。

Skill（技能）

可复用、受约束的执行基元（RSS 管理与认证的对象）。

Task（任务）

面向目标的流程，通常需要在计划下组合多个技能。

Program（程序）

实现代码，可能包含大量临时逻辑、胶水代码与环境假设。

结论

RSS 的重点是把 Program 里的可复用片段抽象成可认证的 Skill，从而让 Task 能被可靠地组合与编排。
