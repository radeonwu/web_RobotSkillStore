概述

RSS-kit 是一个 Python 包，将 RSS 平台 REST API 封装为命令行界面与代码生成器。它处理认证、请求格式化与响应解析——让你的工作流保持在终端中，无需打开浏览器。

  • 脚手架——从模板生成完整的技能规范骨架与可选 ROS 2 包。
  • 本地校验——在接触平台之前在本地运行 Schema 与约束检查。
  • 提交运行记录——一条命令发送执行结果、附上制品、触发治理状态更新。
  • 拉取规范——从平台获取任意已注册技能规范，用于本地开发或二次开发。

安装

pip install rss-kit

需要 Python 3.9+。所有平台命令需要可访问的 RSS 平台实例（自托管或云端）。

三步快速上手

rss-kit · 三步工作流
1
创建技能脚手架
rss-kit new peg_in_hole --template assembly --robot UR10e

生成 peg_in_hole/，包含预填写的 manifest.json、参数默认值及 ROS 2 包骨架。

2
提交前本地校验规范
rss-kit validate peg_in_hole/manifest.json

在本地运行完整的 RSS Schema 检查——在发起网络请求之前发现接口不匹配、缺失约束和错误 Schema 缺口。

3
提交执行运行记录
rss-kit submit --skill peg_in_hole --version 1.0.0 \ --outcome SUCCESS --env real_robot \ --platform-url http://localhost:8080

向平台发送运行记录。提交后治理快照自动重新计算。

命令参考

rss-kit new

从内置模板创建新技能目录。

rss-kit new <skill_id> [OPTIONS]

Options:
  --template   assembly | manipulation | inspection | dispensing | welding
  --robot      UR5e | UR10e | KR10 | IRB1200 | LRMate200iD | ...
  --ros2       同时生成 ROS 2 包骨架（默认：启用）
  --out        输出目录（默认：./）

rss-kit validate

本地对清单进行 RSS 技能规范 Schema 检查，不访问平台。

rss-kit validate <manifest.json> [OPTIONS]

Options:
  --strict     将警告视为错误
  --verbose    显示完整字段级校验报告

rss-kit register

在平台注册已校验的技能规范（POST /skills），状态设置为 REGISTERED。

rss-kit register <manifest.json> --platform-url <url>

rss-kit submit

提交执行运行记录（POST /runs），可选附上制品文件。

rss-kit submit [OPTIONS]

必填参数：
  --skill          skill_id
  --version        语义化版本号（如 1.0.0）
  --outcome        SUCCESS | FAILURE
  --platform-url   RSS 平台基础 URL

可选参数：
  --env            执行环境标签（如 real_robot, simulation, hil）
  --constraint     违反的约束 id（outcome 为 FAILURE 时必填）
  --reason         人类可读的失败说明
  --artifact       制品文件路径（可重复，上传至运行记录）

rss-kit pull

从平台获取已有技能规范，用于本地开发或二次开发。

rss-kit pull <skill_id> [--version <semver>] --platform-url <url>

rss-kit status

打印技能版本的当前生命周期状态与治理快照。

rss-kit status <skill_id> --version <semver> --platform-url <url>

配置文件

将平台 URL 和凭证保存至 ~/.rss-kit/config.toml，避免每次都传入 --platform-url。

[platform]
url     = "http://localhost:8080"
api_key = ""          # 本地无认证实例留空

命令行参数始终覆盖配置文件中的值。

与生命周期的对应关系

RSS-kit 命令直接映射到 RSS 技能生命周期的各个状态转换：

命令 平台接口 目标状态
rss-kit register POST /skills REGISTERED
rss-kit submit（多次运行） POST /runs VALIDATED → CERTIFIED（达到阈值）
rss-kit submit（失败运行） POST /runs DEGRADED（成功率低于阈值时）
rss-kit status GET /skills/{id}/{ver}/governance 只读

完整示例：peg-in-hole 提交流程

# 1 创建脚手架
rss-kit new peg_in_hole --template assembly --robot UR10e

# 2 编辑 manifest.json 后本地校验
rss-kit validate peg_in_hole/manifest.json

# 3 在平台注册
rss-kit register peg_in_hole/manifest.json --platform-url http://localhost:8080

# 4 提交成功运行记录（每次执行后重复）
rss-kit submit --skill peg_in_hole --version 1.0.0 \
  --outcome SUCCESS --env real_robot \
  --artifact peg_in_hole/run_001_telemetry.json \
  --platform-url http://localhost:8080

# 5 查看治理快照
rss-kit status peg_in_hole --version 1.0.0 --platform-url http://localhost:8080
