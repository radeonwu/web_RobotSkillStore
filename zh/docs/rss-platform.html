RSS 平台摘要

RSS 是产品与平台：把“技能”当作可部署系统资产来注册、运行、评测与治理。

返回文档中心 平台总览页

平台能力（4 件事）

注册

管理 Skill / Version / Artifacts，形成可追溯的“发布物”。

运行

所有执行都作为 Run 被平台编排，绑定版本与 Profile，并生成证据。

评测

用 Benchmark 定义成功与稳定性，避免“主观 Demo”。

治理

Policy 门禁在运行时强制：授权、审批、安全、合规不可绕过。

与 RSP / 智能体的关系

  • RSP：RSS 平台强制执行的协议层，保证对象、语义与证据一致。
  • 智能体：RSS 平台内部的“决策层”，只负责提出 Run 请求，不能绕过门禁。
