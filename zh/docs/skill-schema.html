技能 Schema（示例）

规范化 Skill Spec 的 YAML / JSON 表达

总览 概念 技能规格 Schema 执行语义 安全模型 Demo 映射

Canonical YAML

skill:
  name: pick_place
  version: 1.0.0
  intent:
    goal: move_object
    description: 从源点抓取物体并放置到目标点
  context:
    robot: ur5
    environment: tabletop
  constraints:
    force_limit: "<= 30N"
    speed_limit: "<= 0.3 m/s"
    workspace: "defined_volume"
  outcomes:
    success:
      criteria: object_at_target
    failure:
      reasons:
        - force_exceeded
        - planning_failed
        - target_unreachable
  evidence:
    required:
      - runtime_log
      - trajectory
      - metrics

说明

  • constraints 是可强制的边界，而不是建议。
  • outcomes 必须可观测，不能依赖阅读实现代码。
  • evidence 让执行可审计、可比较、可复现。
