RSS 架构摘要

平台为中心，协议强制；智能体只做决策，执行真相由平台持有。

分层与职责

  • Website：面向人（展示、查询、发起请求），不拥有“执行真相”。
  • RSS Platform：注册/版本/运行编排/评测/治理的系统中心。
  • RSP：RSS 强制执行的不变量层（对象、语义、门禁、证据）。
  • Agents：平台内部决策层，通过提交 Run 请求驱动流程。

执行真相与证据

  • 任何动作都必须表现为 Run（平台原子记录）。
  • Run 绑定 Skill Version + Profile，输出 Evidence（日志/轨迹/视频等）。
  • 成功与否由 Benchmark 判定，而非 UI “看起来像”。
