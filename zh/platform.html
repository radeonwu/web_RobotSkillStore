平台

Skill · Agent · Runtime —— 统一的执行平台

Robot Skill Store 并不是零散工具集合，而是围绕技能、Agent 与运行时构建的统一平台。三者共同定义机器人能力如何被创建、选择与执行。

技能：一等公民的能力对象

传统系统中的“技能”常常隐藏在实现里，难以被系统理解与复用。Robot Skill Store 将技能提升为显式的、可共享、可推理的能力对象。

每个技能清晰声明意图、约束、执行上下文与可观测结果，从而具备可发现、可比较、可验证与可复用的属性。

Agent：能力编排者，而非胶水代码

随着系统复杂度提升，依赖硬编码来决定“下一步做什么”会迅速变得脆弱。Agent 的职责是基于目标与上下文选择技能、在候选技能之间排序与权衡、在约束条件下组合多步骤行为，并在不确定性下做出策略性决策。

关键转变

从“我该调用哪个函数？”转向“在当前条件下，哪个技能最适合我的目标？”

运行时：具备语义的执行环境

执行不仅是“下发控制指令”。运行时负责将技能规格绑定到具体机器人或仿真器，强制安全约束，处理环境/硬件差异带来的参数变化，并结构化记录执行结果与反馈。

同一个技能定义可以在仿真与真实之间迁移，而无需改变其语义。这种解耦是实现 sim-to-real 一致性的关键。

技能生命周期（RSP）

  1. 注册：发布元数据、约束与依赖
  2. 发现：按目标/环境检索可用技能
  3. 匹配与排序：Agent 在策略与约束下选择最优技能
  4. 执行：运行时绑定参数并强制约束
  5. 反馈：记录结果，用于后续选择与改进

这一生命周期由 RSP 统一规范，是平台规模化扩展的基础。
