架构

以平台为中心，以协议强制为底座

Robot Skill Store 的架构基于一个明确选择：平台是系统的中心。RSS 平台负责执行、评测与治理，并强制遵循共享协议（RSP），使工具、仿真器、机器人与 Agent 能在不改语义的前提下互操作。

RSP：架构主干

Robot Skill Protocol（RSP）标准化技能描述、发现/选择、约束强制与结果上报。正因为这些交互被统一：

  • 技能可以跨系统迁移
  • Agent 可脱离实现细节进行推理
  • 执行环境可自由替换（仿真器/真实机器人/不同栈）

RSP 不是内部 API，而是支撑生态协作的架构主干。

能力与实现的解耦

Robot Skill Store 明确分离：技能语义（做什么）、实现方式（如何做）与执行环境（在哪里做）。这使得技能与硬件可以独立演进，行为在部署前即可被验证，系统规模化时也能保持一致性。

仿真与现实：同一闭环

仿真不是“测试之后的附加步骤”，而是执行语义的一部分。同一技能规范贯穿仿真验证、测试执行与真实部署：

闭环

定义 → 仿真 → 验证 → 执行 → 反馈

在仿真中不可靠的技能，不应被信任于真实世界。

安全在架构中的位置

安全不是外部附加层，而是被嵌入技能定义、执行约束与运行时强制机制中。将安全前移到技能契约与运行时策略门禁（协议强制），可显著减少对临时补丁与人工兜底的依赖。

为扩展而生

由于 RSS 平台以 RSP 作为稳定底座并强制执行，新机器人/新仿真器/新 Agent 都可以在不重构系统的前提下接入。这正是 Robot Skill Store 能成为长期生态基础的原因。
