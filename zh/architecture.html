平台中心设计

Robot Skill Store 的架构基于一个明确选择：平台是系统的中心。

RSS 平台负责执行、评测与治理，并强制遵循共享协议（RSP），使工具、仿真器、机器人与 Agent 能在不改语义的前提下互操作。

平台中心架构

RSP：架构主干

Robot Skill Protocol（RSP）标准化技能描述、发现/选择、约束强制与结果上报。正因为这些交互被统一：

  • 技能可以跨系统迁移
  • Agent 可脱离实现细节进行推理
  • 执行环境可自由替换（仿真器/真实机器人/不同栈）

RSP 不是内部 API，而是支撑生态协作的架构主干。

能力与实现的解耦

Robot Skill Store 明确分离：技能语义（做什么）、实现方式（如何做）与执行环境（在哪里做）。

这使得技能与硬件可以独立演进，行为在部署前即可被验证，系统规模化时也能保持一致性。

仿真与现实：同一闭环

仿真不是"测试之后的附加步骤"，而是执行语义的一部分。

闭环
定义 → 仿真 → 验证 → 执行 → 反馈
同一技能规范贯穿所有阶段。
🛡️

安全优先设计

安全被嵌入技能定义、执行约束与运行时强制机制中。协议强制门禁减少对临时补丁与人工兜底的依赖。

🔌

为扩展而生

新机器人/新仿真器/新 Agent 都可以在不重构系统的前提下接入。RSP 提供稳定协议底座，支撑长期生态增长。
