Agent*Skill EN
架构

分层解耦，
面向互操作与规模化。

RSP 通过技能接口、编排层与安全运行时，把 Agent 推理与机器人控制解耦。 支持组合、回退、可验证运营，并兼容异构机器人与软件生态。

Agent 负责决策 编排 负责组合 运行时 负责执行 机器人/仿真 负责动作
调用路径
Agent → Skill.call() → 编排器选择实现 → 运行时安全守护 + 监控 → 机器人/仿真执行

技能选择基于机器人能力画像（Capability Profile）与技能约束（Constraints）。

四层架构

Agent 层

LLM/VLA/Policy 等规划器：决定“做什么”，而不是“怎么驱动电机”。

编排层

技能发现、匹配、组合、重试、回退与恢复策略。

运行时层

安全执行：资源限制、安全守护、遥测与统一错误语义。

机器人/仿真层

ROS2/MoveIt/Nav2/Isaac 等仍在底层，通过适配器封装为技能。

协议原语

  • Skill.register：发布 manifest + 运行时端点 + 验证产物
  • Skill.discover：按能力/约束/领域/信任等级检索
  • Skill.call：支持 sync/async/stream，含超时与策略
  • Telemetry：统一日志/指标/链路追踪
互操作
RSP 旨在与既有生态共存。保留 ROS/Isaac 技术栈，RSP 统一外部契约与语义。
