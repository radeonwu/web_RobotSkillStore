把技能做成"可信、可复用"的资产

了解平台 浏览技能

概述

RSS 将机器人技能视为能力资产，并用生命周期管理：注册 → 验证 → 认证 → 部署。

  • 注册：供应方发布技能（接口 + 约束）。
  • 验证：RSS 执行技能并记录运行（runs）。
  • 认证：RSS 基于证据与指标形成认证快照。
  • 部署：用户复用已认证技能，获得可预测的交付行为。

注册 Register

目的 本页说明技能供应方如何将机器人技能提交到 RSS。

你将看到什么 你会了解技能接口是什么样、必须声明哪些约束，以及在技能进入验证前 RSS 需要哪些关键信息。

生命周期锚点 RSS 技能生命周期 → 注册（供应方 → RSS）

  • 什么是技能注册：技能包 + 元数据 + 接口契约。
  • 技能接口与约束：明确前置条件、安全限制与输入输出规范。

验证 Validate

目的 本页介绍 RSS 如何在受控环境中执行机器人技能。

你将看到什么 你会看到技能如何被运行、执行结果如何被记录，以及 RSS 如何验证正确性、稳定性与可重复性。

生命周期锚点 RSS 技能生命周期 → 验证（RSS 内部）

  • 技能执行与仿真：仿真 / 硬件在环运行。
  • 运行记录：每次运行都有可追溯记录与上下文。

认证 Certify

目的 本页说明 RSS 如何评估并认证机器人技能。

你将看到什么 执行日志、指标与各类产物会被收集，用于评估技能质量与部署就绪度。

生命周期锚点 RSS 技能生命周期 → 认证（RSS 内部）

  • 证据与指标：成功率、时延、安全约束、鲁棒性等。
  • 工件管理：日志、视频、数据集、报告。
  • 认证逻辑：策略 + 阈值 + 治理。

部署 Deploy

目的 本页展示已认证技能如何交付给用户使用。

你将看到什么 你会了解已验证技能如何被打包、部署，并在不同机器人、项目与场景中复用。

生命周期锚点 RSS 技能生命周期 → 部署（RSS → 用户）

  • 技能打包：可复现的 bundle + 依赖锁定。
  • 部署场景：产线、移动操作、户外机器人等。
