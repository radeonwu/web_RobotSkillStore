用户与部署

目的把认证技能变成可复用的现场应用。

本页内容以 UR5/RG2 示例说明技能如何复用；并推荐使用“智能体编排”提升效率：规划 → 匹配 → 执行 → 失败恢复。

生命周期锚点

RSS 面向用户端输出“技能资产”，用于快速拼装应用流程，而不是一次性 demo 代码。

基于智能体的技能部署流程

应用团队通常从用户任务出发（例如，使用 UR5 机械臂和 RG2 夹爪完成连接器装配并拧紧螺丝），由智能体把认证技能当作积木进行编排与执行。

智能体在做什么（可操作的闭环）

  • 抓取：根据工件位姿与工作空间约束，为 pick_place 选择抓取模板、接近高度、避碰策略。
  • 插入：为 connector_insert 设置位姿约束、接近/后撤参数与失败恢复阈值（接触/入座/退出）。
  • 拧紧：为 tighten_screw 配置 torque/angle 任务，并用证据验证确定性与一致性。
  • 治理：每次运行都形成 trace 包；稳定性通过“验证→认证”提升，而不是靠反复改 demo 代码。
// 伪代码：用智能体对认证技能做编排
goal = "assemble_connector_and_tighten"
plan = ["pick_place", "connector_insert", "tighten_screw"]

for step in plan:
    candidates = rss.search(skill_name=step, evidence_min="E1")
    skill = agent.select_best(candidates, context)
    params = agent.configure(skill, context, constraints)
    run = rss.execute(skill, params)
    if not run.success:
        context = agent.recover(run, context)  # 重试 / 调参 / 换技能
        continue
    agent.update_context_from_evidence(run.evidence)

连接器装配应用示例

示例流程：pick_place → connector_insert → tighten_screw。同一套技能通过调整约束与参数即可复用，无需重写程序。

  • pick_place：抓取模板 + 避碰 IK。
  • connector_insert：位姿约束 + 接近/后撤 + 失败恢复 trace。
  • tighten_screw：确定性拧紧，并输出 torque/angle 证据。

为什么智能体编排很重要

智能体编排不只是“建议”——它决定了应用团队能否规模化交付。智能体自动化 plan/match/execute/recover 的集成工作，而 RSS 让每个技能保持模块化、证据可追溯、可治理。

  • 更快迭代：失败变成“调参 + 复盘证据”，而不是反复改流程代码。
  • 更安全复用：约束与成熟度门禁降低跨工位/跨项目的部署风险。
  • 更可迁移：同一计划可通过替换技能绑定，迁移到不同工站与设备。

建议路径

从 认证技能 开始，进入 证据查看器 查看 runs/工件；在用户侧通过参数与约束复用技能；再通过验证→认证推进成熟度。
