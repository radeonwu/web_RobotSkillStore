## What Changed in v6.80

### 1. User Authentication System

v6.80 introduces a three-role authentication layer connected to the RSS Platform backend.

| Role | Access | How to obtain |
|------|--------|---------------|
| **Visitor** | Browse skill catalog, read docs | No login required |
| **End User** | Browse + download skill bundles | Register account (JWT) |
| **Developer** | All above + Platform & Developers pages | Register with invite code (JWT) |

**Auth modes:**

- **JWT mode** (recommended): `RSS_PLATFORM_BASE_URL` is set in `assets/config.js`. Registration and login call the platform `/auth/` API. Tokens are stored in `localStorage`.
- **Passkey mode** (offline / `file://`): No platform URL set. Access is granted by entering the correct passkey configured in `assets/config.js`.

### 2. Skill Download Gate

Downloading a skill bundle from the Skill Catalog page (`en/skills.html`, `zh/skills.html`) now requires End User or Developer login. Visitors see a sign-in prompt when clicking "Download bundle".

### 3. Sign-In / Register UI

- Account button in the top-right nav shows role badge when logged in.
- Register form includes role selection (End User / Developer).
- Developer registration requires an invite code.
- "Continue as Visitor" option always available.

## Configuration Reference

### Website — `assets/config.js`

```javascript
// Point to your running platform instance (no trailing slash)
window.RSS_PLATFORM_BASE_URL = "http://localhost:8080";

// Passkey mode fallback credentials (offline / file:// access)
window.RSS_USER_PASSKEY = "rss-user-2026";   // end-user passkey
window.RSS_DEV_PASSKEY  = "rss-dev-2026";    // developer passkey
```

### Platform — `docker-compose.yml` (api1 / api2 env)

| Variable | Default | Purpose |
|----------|---------|---------|
| `RSS_JWT_SECRET` | `rss-jwt-secret-change-in-prod` | HMAC key for signing JWT tokens — **change before production** |
| `RSS_JWT_EXPIRY_DAYS` | `7` | Token lifetime in days |
| `RSS_DEV_INVITE_CODE` | `rss-dev-invite-2026` | Code required to register a Developer account |

## Auth API Endpoints (Platform)

All endpoints are at `/auth/` and routed through the nginx gateway on port 8080.

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| `POST` | `/auth/register` | None | Register new account |
| `POST` | `/auth/login` | None | Login, returns JWT token |
| `GET`  | `/auth/me` | Bearer JWT | Get current user info |
| `GET`  | `/auth/admin/users` | X-API-Key (admin) | List all users |
| `PATCH`| `/auth/admin/users/{id}` | X-API-Key (admin) | Update user role / active |
| `DELETE`| `/auth/admin/users/{id}` | X-API-Key (admin) | Delete user |

**Register payload example:**
```json
{
  "name": "Alice",
  "email": "alice@example.com",
  "password": "securepassword",
  "role": "enduser"
}
```
For role `"developer"`, add `"invite_code": "rss-dev-invite-2026"`.

## Connecting Website to Platform

1. Start the platform:
   ```bash
   cd /path/to/RSS_Platform_v1.0.10
   docker compose up -d
   ```

2. Confirm the platform is running:
   ```bash
   curl http://localhost:8080/health
   # → {"status":"ok"}
   ```

3. Set the platform URL in the website config:
   ```javascript
   // assets/config.js
   window.RSS_PLATFORM_BASE_URL = "http://localhost:8080";
   ```

4. Open the website. Skills load live from the platform. Auth features activate automatically.

## Known Constraints

- The website must be served over HTTP (not `file://`) for the platform API fetch to work reliably. Use `python3 -m http.server 8000` or place behind the nginx gateway.
- `RSS_JWT_SECRET` and `RSS_DEV_INVITE_CODE` must be identical across `api1` and `api2` containers.
- Change `RSS_JWT_SECRET` before any production deployment.
