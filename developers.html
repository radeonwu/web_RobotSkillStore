Developer workflow

开发者工作流示意

1. Design skill behavior
   设计技能行为（Pick-Place / Screw-Tightening / 插接等）

2. Define skill schema & capability requirements
   定义技能的输入/输出 Schema 与能力需求

3. Implement using RSP SDK + ROS2 / OEM SDK
   通过 RSP SDK + ROS2 / 厂商 SDK 实现技能逻辑

4. Test in local sandbox & simulation
   在本地沙盒与仿真环境中测试

5. Submit for validation & publishing
   提交平台进行安全与兼容性验证，上架技能商店

6. Monitor usage & iterate based on feedback
   通过使用数据与反馈持续迭代技能版本
