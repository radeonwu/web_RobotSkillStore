## What Changed in v6.82

### 1. Companion Domain — Skill Catalog Support

The website now fully supports the DESKMATE companion skill domain introduced in RSS Platform v1.1.0.

**`en/skills.html` and `zh/skills.html`:**
- `normalizeFromPlatform()` now passes through `domain`, `scene`, `device`, `context_sources`, `output_channels`, `privacy_mode` from both the live platform API and the static catalog.
- `renderSkill()`: companion cards show device / scene / privacy_mode badges and a `context_sources` line instead of robot badges.
- `renderDeployPanel()`: companion skills show `deskmate-kit install` + `deskmate-cli skill activate` + evidence run payload; industrial skills unchanged.
- `triggerDownload()`: companion branch generates a v2.2 `deskmate_skill_pack` manifest (`agent` / `trigger` / `requirements` structure); industrial branch continues to generate RSS Manifest v2.1.
- `INTENT_MAP`: 7 companion intent entries added (EN + ZH keywords for focus, study, writing, gaming, companion, reading, deskmate).
- `CAT_DEF`: `'Companion'` entry added (risk: LOW, payload: 0).

**`assets/data/public_skills.json` and `public_skills.js`:**
- 6 DESKMATE companion skills added (`focus_mode`, `study_buddy`, `writing_coach`, `gaming_companion`, `daily_companion`, `reading_aide`).
- Total catalog: 74 industrial + 6 companion = **80 skills**.

### 2. Broader Domain Framing — Home & Catalog Pages

The site identity has been updated from "industrial robot skills" to "robot skills across every domain", reflecting the platform's expansion to companion and future skill types.

**`en/index.html`:**
- Page title: "Trusted **Industrial** Robot Skills" → "Trusted **Robot** Skills"
- Hero headline: "Deploy robot skills like industrial apps" → "One store for every kind of robot skill"
- Hero tagline: now includes "industrial automation, desktop companions, and beyond"
- Stats bar: 68 / 9 categories / 10+ robot models → **80 / 10 categories / 2 domains**
- Hero summary cards: broadened to domain-agnostic language
- Category subtitle (static + dynamic JS): shows `N skills across X categories (M industrial · K companion)`
- Static category grid: **Companion** card added (pink, 6 skills)
- "View all" link: 68 → **80**
- Applications section: "Applications in the Field" → "Applications Across Domains"; subtitle broadened; **DESKMATE Focus & Study Session** app card added as 5th card

**`zh/index.html`:** same changes in Chinese:
- Title: "可信**工业机器人**技能平台" → "可信**机器人**技能**开放**平台"
- Hero: "可信**工业机器人**技能的标准化平台" → "**机器人技能**的标准化**开放**平台"
- Hero tagline: "工业自动化、桌面陪伴，乃至更多机器人应用场景"
- Stats bar: 68 / 9 / 10+ → **80 / 10 / 2 领域**
- Category subtitle: "68 个工业机器人技能，覆盖 9 大制造领域" → "80 个机器人技能，覆盖 10 大类别（工业 · 陪伴 · 持续扩展）"
- Category grid: 桌面陪伴 card added
- Applications: "实际应用场景" → "**跨领域**应用场景"; DESKMATE companion app card added

**`en/skills.html` and `zh/skills.html`:**
- Catalog h1: "Find installable robot skills by task, fit, and trust" → "Find deployable robot skills **across every domain**"
- Description: explicitly mentions "industrial automation, desktop companions, and more"
