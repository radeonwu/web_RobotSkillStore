Platform

The Platform executes and governs skills (projects, quotas, audit). This Website hosts concept docs and demos.

Skill · Agent · Runtime — A Unified Execution Platform

Robot Skill Store is not a collection of tools. It is a platform designed around three first-class citizens: Skills, Agents, and Runtime. These elements are not optional modules — they define how robot capabilities are created, selected, and executed.

Skills as First-Class Capabilities

In traditional robotics systems, skills are buried inside codebases, tightly coupled to hardware, parameters, and environments. Robot Skill Store treats skills as explicit, shareable capability objects.

Each skill:

  • declares its intent and assumptions
  • specifies constraints and safety boundaries
  • defines required execution context
  • exposes observable outcomes and feedback

This makes skills discoverable, comparable, executable across simulation and real robots, and reusable beyond their original implementation. A skill is no longer just something you call — it is something the system can reason about.

Agents as Orchestrators, Not Glue Code

As robotic systems grow in complexity, hard-coded logic becomes brittle. Robot Skill Store introduces agents as first-class orchestrators.

Agents are responsible for:

  • selecting skills based on goals and context
  • ranking alternatives under uncertainty
  • resolving trade-offs between constraints
  • composing multi-step behaviors
Shift in mindset

From “Which function should I call?” to “Which skill best satisfies my intent under current conditions?”

Runtime as a Semantic Execution Environment

Execution is not just sending commands. The runtime binds skill specifications to concrete robots and simulators, enforces constraints during execution, handles environment-specific parameters, and collects execution feedback.

The same skill can be executed in simulation, on different robots, and under different runtime configurations without changing its semantic definition. This decoupling enables simulation-to-real consistency, reproducible demos, and safe experimentation.

The Skill Lifecycle

The platform is organized around a clear and explicit lifecycle:

  1. Register — define and publish metadata and constraints
  2. Discover — agents search and evaluate candidates
  3. Match & Rank — select based on intent, context, and policies
  4. Execute — runtime binds and executes safely
  5. Feedback — outcomes inform future decisions

This lifecycle is formalized by RSP and shared across the platform, enabling scalable capability sharing, agent-driven orchestration, and ecosystem-level interoperability.
