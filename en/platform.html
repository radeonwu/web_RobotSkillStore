Platform Runtime Pipeline

This pipeline is taken from the current RSS Platform stack (v1.0.4.12). Validation produces Runs + Evidence; certification computes governance state from those immutable records.

UI entrypoints (via Gateway): /viewer/ for evidence browsing, and /public/v1/* for website embedding.

Core Platform Components

Building blocks for trusted robot skills

Skill Registry

Publish skills as stable capability contracts (schema, constraints, versioning). Specs are schema-validated before acceptance.

  • Validate spec on ingest: JSON Schema (Draft 2020-12) via validate_skill_spec()
  • Create / list skills: POST /skills, GET /skills, GET /skills/{skill_id}
  • Taxonomy binding: GET /skills/{skill_id}/{version}/taxonomy

Validation Engine

Every execution becomes a platform-managed Run with immutable context and evidence.

  • Run creation: POST /runs (or POST /runs/submit)
  • Evidence ingestion: POST /runs/{run_id}/artifacts
  • Trace & events: GET /runs/{run_id}/events
  • Read model: GET /runs/{run_id} is source of truth

Evidence & Artifact Store

Artifacts stored on local filesystem or S3/MinIO. Every artifact linked to a Run for auditability.

  • Upload artifacts: POST /runs/{run_id}/artifacts
  • List artifacts: GET /runs/{run_id}/artifacts
  • Fetch / download: GET /artifacts/{artifact_id}
  • Presigned URLs: GET /artifacts/{artifact_id}/url

Certification & Governance

Evidence-driven governance state computed from historical Run windows using explicit rules.

  • Governance snapshot: GET /skills/{skill_id}/{version}/governance
  • Policy evaluation: POST /governance/policies/eval
  • Public rollups: GET /public/v1/skills
  • Admin refresh: POST /admin/public/v1/rollup

Auth, Anon-Read, and API Keys

RSS Platform separates public read (for websites and sharing) from authenticated operations (write, execution, admin). The switch is controlled by RSP_AUTH_ALLOW_ANON_READ.

  • Public read path: when enabled, websites can read aggregated skill metadata via /public/v1/*
  • Authenticated path: non-public endpoints require X-API-Key: <key> header
  • Default posture: keep anon-read off in production unless intentionally exposing public stats

Environment flags

RSP_AUTH_ALLOW_ANON_READ=1    # allow GET /public/v1/* without credentials
X-API-Key: <key>                 # required for non-public routes

Rule of thumb: /public/v1 is for reading curated summaries; everything that mutates state stays behind auth.

Artifact Store Backends

The artifact store is pluggable. The platform selects a backend with RSP_ARTIFACT_STORE and serves downloads either directly (local) or via presigned URLs (S3/MinIO).

  • Local filesystem: artifacts stored on the API/worker host; GET /artifacts/{artifact_id} streams bytes through the platform
  • S3/MinIO: artifacts stored as objects; platform issues time-limited presigned URLs for upload/download
  • Presign flow: UI requests URL ‚Üí platform signs ‚Üí browser/worker transfers bytes directly to storage

Why presign: artifact traffic can be GB-scale. Presigned URLs keep the control plane lightweight while preserving traceability.
