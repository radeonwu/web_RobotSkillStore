Platform Console

Operate the trust and evidence layer behind the store

The platform is the operational side of Robot Skill Store. It manages runs, artifacts, lifecycle evaluation, and the public surfaces that turn raw robotics work into a credible skill marketplace.

What the Platform Does

These are the backend capabilities that make the public skill storefront believable instead of cosmetic.

🔍
Skill Registry
Accepts structured skill specs (JSON Schema validated), assigns stable IDs, and makes skills searchable in the public catalog immediately.
▶
Run Management
Every execution creates an immutable Run record — bound to skill version, robot profile, and parameter set. Outcomes, constraints, and reasons are indexed.
📦
Artifact Storage
Logs, videos, and telemetry are stored (local disk or S3/MinIO) and linked to their Run. Immutable after upload. Accessible via the Evidence Viewer.
⚙
Governance Engine
Computes ACTIVE / DEGRADED / QUARANTINED from rolling evidence windows — success rate, violation rate, regression signals. No manual assignment.
🏆
Certification
When evidence meets policy thresholds, a governance snapshot is issued and the skill advances to CERTIFIED — published publicly with its evidence trail.
🔒
Auth & Access
Public read routes (/public/v1/skills) work without credentials. Write and admin operations require an API key. Anon-read is configurable per deployment.

Evidence Viewer

The Evidence Viewer is the operator-facing console: the place users go after discovery, once they need to inspect run history, artifacts, and governance state in detail.

  • Browse all runs for any skill version — filter by outcome, robot, date.
  • View attached artifacts: video playback, log inspection, telemetry charts.
  • Inspect governance snapshots: what passed, what triggered DEGRADED, and why.
