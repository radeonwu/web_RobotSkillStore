Stage 2 of 4 · Lifecycle

Validate

Build an evidence base by running the skill on real hardware, HIL, or simulation — every execution is recorded and immutably stored.

What happens

  • Submit runs via POST /runs — outcome, constraint violations, and reason recorded per execution.
  • Attach artifacts — logs, video clips, and telemetry data linked immutably to each run; cannot be modified after upload.
  • Evidence accumulates — RSS tracks a rolling window of success rate and violation rate across all submitted runs.

What the platform does

  • Indexes each run record — searchable by outcome, robot, date, and constraint state.
  • Stores artifacts on local disk or S3/MinIO — binds them to the run and marks them immutable.
  • Begins computing governance metrics — success rate, violation rate, and regression signals update after each run.
