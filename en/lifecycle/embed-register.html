Register

Publish a structured skill spec — the formal contract that declares what a skill does, what it requires, and how it can fail.

What happens

  • Submit a skill spec via POST /skills — package, metadata, interface contract, and error schema in one JSON document.
  • Declare constraints — preconditions, safety limits, and IO schema are validated against the RSS JSON Schema on ingest.
  • Receive a stable skill ID — the skill is immediately discoverable in the public catalog with status REGISTERED.

What the platform does

  • Runs schema validation — rejects malformed or incomplete specs before storage.
  • Assigns an immutable skill_id tied to the skill name and version.
  • Makes the skill searchable by category, capability, and robot compatibility.
