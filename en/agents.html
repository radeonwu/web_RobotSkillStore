What is an Agent (in RSS)

An agent is an orchestrator that reasons over goals, context, and historical evidence to choose which Skill Version to run, with which Profile and inputs.

In Robot Skill Store (RSS), agents are a decision layer for selecting and composing skills. They never bypass governance, evaluation, or execution truthâ€”every action becomes a platform-managed Run with immutable evidence.

Where Agents Sit in the System

Agents operate at the decision layer, while RSS handles enforcement and truth. This separation ensures:

  • Policy gates: what is allowed (safety, approval, authorization)
  • Version binding: every Run binds to immutable Skill Version + Profile
  • Objective evaluation: benchmark-defined success enables comparability
  • Auditability: evidence is first-class; decisions are explainable and reviewable
