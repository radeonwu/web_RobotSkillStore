Canonical YAML

skill:
  name: pick_place
  version: 1.0.0
  intent:
    goal: move_object
    description: Pick an object from source and place it at target
  context:
    robot: ur5
    environment: tabletop
  constraints:
    force_limit: "<= 30N"
    speed_limit: "<= 0.3 m/s"
    workspace: "defined_volume"
  outcomes:
    success:
      criteria: object_at_target
    failure:
      reasons:
        - force_exceeded
        - planning_failed
        - target_unreachable
  evidence:
    required:
      - runtime_log
      - trajectory
      - metrics

Notes

  • constraints are enforceable boundaries, not suggestions.
  • outcomes must be observable without reading implementation code.
  • evidence makes runs auditable and comparable.
