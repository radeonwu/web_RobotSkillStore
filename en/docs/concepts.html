Concepts

The minimal abstractions of RSP

Overview Concepts Skill Spec Schema Execution Safety Demo Mapping

Skill

A Skill is the smallest meaningful and safe unit of robotic capability. It is defined by intent, constraints, context, and observable outcomes — not by code alone.

Agent

An Agent selects, composes, and plans skills based on goals, policies, and risk tradeoffs. Agents reason over specifications.

Runtime

The Runtime binds abstract skill specifications to concrete systems (robot, TCP, environment) and enforces constraints during execution.

Evidence

Evidence includes logs, metrics, trajectories, and outcomes. Evidence enables auditability, evaluation, and reuse.
