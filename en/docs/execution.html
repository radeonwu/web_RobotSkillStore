Execution Semantics

From spec to a reproducible, auditable run

Overview Concepts Skill Spec Schema Execution Safety Demo Mapping

Pre-execution validation

Before execution, the runtime validates that required context is present and constraints are enforceable. If not, it fails fast with evidence.

Runtime binding

Binding maps abstract spec fields to concrete system parameters:

  • Robot base pose
  • TCP / tool frame
  • Environment frames, fixtures, and calibration

Constraint enforcement

Constraints must be enforced during execution (e.g., force/speed/workspace). Enforcement is a runtime responsibility, not an implementation promise.

Outcomes and Evidence

Each run produces (1) an outcome (success/failure) and (2) evidence (logs/metrics/trajectory). Evidence makes runs auditable and comparable.
