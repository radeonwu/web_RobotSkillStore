RSS Architecture Summary

Platform-centered, protocol-enforced: RSS is the center; RSS enforces RSP invariants at runtime.

Core layers
  • Control plane: skill registry, versions, approvals, policy.
  • Data plane: runs, evidence capture, metrics, artifacts.
  • Agent plane: planners that propose Runs; never bypass gates.
RSP’s role
  • Canonical objects: Skill / Run / Profile / Policy / Evidence.
  • Invariants: binding, enforcement, auditability.
  • Portability: consistent meaning across sim and real robots.
