Overview

RSS-kit is a Python package that wraps the RSS Platform REST API with a command-line interface and a code generator. It handles authentication, request formatting, and response parsing — so your workflow stays in the terminal, not a browser.

  • Scaffold — generate a complete skill spec skeleton and optional ROS 2 package from a template.
  • Validate locally — run schema and constraint checks before touching the platform.
  • Submit runs — post execution outcomes, attach artifacts, and trigger governance recomputation in one command.
  • Pull specs — fetch any registered skill spec from the platform for local development or forking.

Installation

pip install rss-kit

Requires Python 3.9+. All platform commands require a reachable RSS Platform instance (self-hosted or cloud).

3-Step Quickstart

rss-kit · 3-step workflow
1
Scaffold a new skill
rss-kit new peg_in_hole --template assembly --robot UR10e

Creates peg_in_hole/ with a pre-filled manifest.json, parameter defaults, and a ROS 2 package skeleton.

2
Validate the spec before submitting
rss-kit validate peg_in_hole/manifest.json

Runs the full RSS schema check locally — catches interface mismatches, missing constraints, and error schema gaps before any network call.

3
Submit an execution run
rss-kit submit --skill peg_in_hole --version 1.0.0 \ --outcome SUCCESS --env real_robot \ --platform-url http://localhost:8080

Posts a run record to the platform. Governance snapshot is recomputed automatically after submission.

Command Reference

rss-kit new

Scaffold a new skill directory from a built-in template.

rss-kit new <skill_id> [OPTIONS]

Options:
  --template   assembly | manipulation | inspection | dispensing | welding
  --robot      UR5e | UR10e | KR10 | IRB1200 | LRMate200iD | ...
  --ros2       also generate a ROS 2 package skeleton (default: true)
  --out        output directory (default: ./)

rss-kit validate

Check a manifest against the RSS skill spec schema without contacting the platform.

rss-kit validate <manifest.json> [OPTIONS]

Options:
  --strict     treat warnings as errors
  --verbose    show full field-level validation report

rss-kit register

Register a validated skill spec on the platform (POST /skills). Sets state to REGISTERED.

rss-kit register <manifest.json> --platform-url <url>

rss-kit submit

Submit an execution run record (POST /runs). Optionally attach artifact files.

rss-kit submit [OPTIONS]

Required:
  --skill          skill_id
  --version        semver string (e.g. 1.0.0)
  --outcome        SUCCESS | FAILURE
  --platform-url   base URL of the RSS Platform

Optional:
  --env            execution environment tag (e.g. real_robot, simulation, hil)
  --constraint     violated constraint id (required if outcome is FAILURE)
  --reason         human-readable failure note
  --artifact       path to artifact file (repeatable; uploads to run record)

rss-kit pull

Fetch an existing skill spec from the platform for local development or forking.

rss-kit pull <skill_id> [--version <semver>] --platform-url <url>

rss-kit status

Print the current lifecycle state and governance snapshot for a skill version.

rss-kit status <skill_id> --version <semver> --platform-url <url>

Configuration

Store your platform URL and credentials in ~/.rss-kit/config.toml to avoid passing --platform-url on every command.

[platform]
url     = "http://localhost:8080"
api_key = ""          # leave empty for unauthenticated local instances

Command-line flags always override the config file.

Lifecycle Integration

RSS-kit maps directly onto the RSS skill lifecycle. Each command targets a specific state transition:

Command Platform call Resulting state
rss-kit register POST /skills REGISTERED
rss-kit submit (×N runs) POST /runs VALIDATED → CERTIFIED (threshold met)
rss-kit submit (failure run) POST /runs DEGRADED (if success rate drops below threshold)
rss-kit status GET /skills/{id}/{ver}/governance read-only

Full Example: peg-in-hole submission

# 1 scaffold
rss-kit new peg_in_hole --template assembly --robot UR10e

# 2 edit manifest.json, then validate
rss-kit validate peg_in_hole/manifest.json

# 3 register on platform
rss-kit register peg_in_hole/manifest.json --platform-url http://localhost:8080

# 4 submit successful runs (repeat after each execution)
rss-kit submit --skill peg_in_hole --version 1.0.0 \
  --outcome SUCCESS --env real_robot \
  --artifact peg_in_hole/run_001_telemetry.json \
  --platform-url http://localhost:8080

# 5 check governance snapshot
rss-kit status peg_in_hole --version 1.0.0 --platform-url http://localhost:8080
