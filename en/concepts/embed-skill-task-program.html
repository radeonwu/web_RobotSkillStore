Skill ≠ Task ≠ Program

RSS separates reusable capabilities from one-off workflows.

Skill

A reusable, constrained execution primitive (what RSS manages and certifies).

Task

A goal-oriented workflow that composes one or more skills under a plan.

Program

Implementation code that may contain ad-hoc logic, glue, and environment assumptions.

Implication

RSS focuses on turning program fragments into certifiable skills, so tasks can be composed reliably by humans or agents.
