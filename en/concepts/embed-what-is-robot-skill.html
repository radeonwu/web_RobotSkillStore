What is a Robot Skill?

A robot skill in RSS is a deterministic, executable capability that controls physical robots under explicit constraints.

Definition

A Robot Skill is an execution primitive that can be validated, certified, and reused across deployments.

  • Executable on real robots
  • Deterministic behavior (within declared constraints)
  • Explicit preconditions, safety limits, and I/O schema
  • Verifiable through evidence artifacts and metrics

Core idea (What): redefining a Robot Skill

A Skill is not just an action script, trajectory, or one-off demo.

A Skill is a callable, evaluable, and transferable capability unit.

A Skill must have

  • Clear input/output semantics (I/O schema and constraints)
  • Implementation freedom: rules, RL, VLA, world model, or hybrids
  • Measurable quality: success rate, applicability domain, and risk boundaries
  • Agent-ready orchestration: schedulable, composable, replaceable

Why RSS cares

When execution cost and risk are high, validation and governance become non-negotiable. RSS exists to make robot skills portable and trustworthy.
