A Robot Skill is an execution primitive that can be validated, certified, and reused across deployments.

Executable on real robots · Deterministic behavior (within declared constraints) · Explicit preconditions, safety limits, and I/O schema · Verifiable through evidence artifacts and metrics

✕ Not just a…

  • Action script or trajectory
  • One-off demo or prototype
  • Hardcoded, non-transferable program

✓ A Skill is…

  • Callable, evaluable, transferable
  • Constrained by explicit contracts
  • Certified by evidence, not reputation
I/O Semantics
Clear input/output schema and constraints define the skill's interface and make it composable across systems.
Implementation freedom
The implementation may use rules, RL, VLA, world models, or hybrids — the interface contract stays fixed.
Measurable quality
Success rate, applicability domain, and risk boundaries are first-class properties, not afterthoughts.
Agent-ready orchestration
Skills are schedulable, composable, and replaceable — enabling safe agent-driven workflows.

When execution cost and risk are high, validation and governance become non-negotiable. RSS exists to make robot skills portable and trustworthy.
