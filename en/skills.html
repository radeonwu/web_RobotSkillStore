Pipeline (example workflow)

Pipeline content has been merged into this page. It shows how multiple skills form an end-to-end workflow.

1) Pick

Grasp an object from a source location with defined approach/attach tolerances.

  • Input: object pose / bin / constraints
  • Output: grasped object with evidence
  • Status: stable; benchmarkable

2) Insert

Align and insert the object into a target socket with contact-aware motion.

  • Input: target pose / clearance / force limits
  • Output: inserted state + contact traces
  • Status: improving; measurable

3) Tighten Screw

Tighten to a torque/angle goal with policy gates and immutable records.

  • Input: screw pose / axis / speed limits
  • Output: tightened result + video/metrics
  • Status: benchmarked; auditable
Tip: In RSS, the pipeline is executed as a sequence of Runs, each bound to an immutable Skill Version and a Profile.
