Safety

Safety is a protocol primitive.

RSP v0.3 treats safety and trust as first-class concerns: skills are capability-bound, simulation-validated, runtime-guarded, and auditable.

Capability-bound Runtime guards Sim-first Audit-ready
Trust model
Skill = Manifest + Implementation + Verification + Telemetry Trust signals: tests, sim reports, runtime policy, version history

Operators choose what trust level a deployment requires (dev / staging / production).

Key mechanisms

Preconditions & constraints

Each skill declares what must be true before execution and the safety envelope it will respect.

Runtime safety guards

Limits on motion, forces, zones, speed, resource use; automatic abort & rollback paths.

Simulation-first validation

Skills ship with tests and sim scenarios to reduce real-world risk and improve reproducibility.

Human-in-the-loop escalation

Clear escalation states for uncertain conditions, manual approvals, or remote operation.

Auditable telemetry

Standard logs/metrics/traces for incident investigation, improvement, and compliance.

Certification-ready artifacts

Optional verification reports that can be attached to skills and enforced by policy.
