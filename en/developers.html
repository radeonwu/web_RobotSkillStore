How to Submit a Skill

Four steps from a robot capability to a platform-certified store listing.

1

Define the Skill Spec

A JSON document declaring intent, interface (inputs/outputs), pre/postconditions, safety constraints, and error schema. The spec is the contract everything else builds on.

2

Register on the Platform

POST /skills — RSS validates schema, assigns a skill_id, and sets state to REGISTERED. Discoverable in the catalog immediately.

3

Run Validation Executions

POST /runs per execution (outcome, constraint, reason). Attach artifacts via POST /runs/{run_id}/artifacts. All records stored immutably.

4

Certification & Governance

GET /skills/{skill_id}/{version}/governance computes a snapshot — success rate, violations, regression. Passing advances state to CERTIFIED and publishes the skill.

What RSS Validates

Validation should strengthen the buyer decision on the storefront. That means RSS checks not just schema correctness, but evidence quality and operational behavior over time.

Validation criteria — 6 items ▾
  • Spec compliance — interface, constraints, and error schema validated on ingest.
  • Execution outcomes — every run records outcome, constraint violated, and reason.
  • Artifact integrity — logs, videos, telemetry linked to their run; immutable after upload.
  • Rolling success rate — evaluated over a window (e.g. last 30 runs), not single shots.
  • Violation rate — hard safety violations tracked separately from soft failures.
  • Regression detection — success rate drops on certified skills trigger DEGRADED automatically.
Evidence levels: as runs accumulate, skills unlock broader deployment permissions and higher trust signals in the catalog.

RSS-kit: Developer Toolkit

RSS-kit is the fastest path from local development to a live store listing. It reduces platform friction so suppliers can focus on capability quality instead of transport boilerplate.

rss-kit · 3-step workflow
1
Install and scaffold a new skill
pip install rss-kit rss-kit new peg_in_hole --template assembly --robot UR10e
2
Validate the spec locally before submitting
rss-kit validate peg_in_hole_manifest.json
3
Submit an execution run to the platform
rss-kit submit --skill peg_in_hole --version 1.0.0 \ --outcome SUCCESS --env real_robot \ --platform-url http://localhost:8080

RSS-kit also scaffolds ROS 2 packages from your spec and can pull existing skill specs from the platform for local development.  RSS-kit Docs

Skill Spec Format

The spec is the contract behind every public skill page, compatibility check, and downloadable bundle.

Skill Spec — minimum structure (JSON) ▾ 
{
  "skill_id": "peg_in_hole",
  "version": "1.0.0",
  "name": "Peg-in-Hole Insertion",
  "category": "Assembly",
  "maturity": "beta",
  "intent": "Insert a peg into a hole with tight tolerance using compliant motion.",
  "interface": {
    "inputs": {
      "hole_pose":   { "type": "pose",  "frame": "world" },
      "peg_length":  { "type": "float", "unit": "mm" },
      "max_force_n": { "type": "float", "unit": "N", "robot_relative": true }
    },
    "outputs": {
      "outcome":          { "type": "string", "enum": ["SUCCESS", "FAILURE"] },
      "insertion_depth":  { "type": "float",  "unit": "mm" }
    }
  },
  "conditions": {
    "pre":  [{ "id": "peg_grasped",   "check": "gripper.closed == true" }],
    "post": [{ "id": "fully_inserted","check": "insertion_depth >= peg_length * 0.95" }]
  },
  "error_schema": {
    "FORCE_EXCEEDED":   { "severity": "CRITICAL", "recoverable": false },
    "SEARCH_EXHAUSTED": { "severity": "HIGH",     "recoverable": true  }
  },
  "deployment": {
    "requires_capabilities": {
      "min_dof": 6,
      "requires_force_torque_sensor": true
    }
  }
}
