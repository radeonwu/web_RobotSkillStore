Developers

This page is intentionally concise: developer onboarding (registration & governance) is covered in Platform. Use this page as a quick entry point.

Robot Skill Store is designed to be adopted incrementally. You do not need to replace your existing robotics stack. This page describes where to start and how to run your first skill using the Robot Skill Protocol (RSP).

What You Need

  • a robot or simulator (e.g., UR5, Gazebo, Isaac Sim)
  • a basic runtime environment (ROS, Python, or equivalent)
  • a skill definition that follows RSP

Robot Skill Store does not impose a specific control stack. It integrates with what you already use.

The Minimal Workflow

At a high level, working with Robot Skill Store follows four steps:

  1. Define a Skill — describe intent, constraints, and outcomes
  2. Register the Skill — publish it so agents can discover it
  3. Select the Skill — agent evaluates candidates based on goal & context
  4. Execute the Skill — runtime binds parameters and enforces constraints

Each step is explicit and observable.

Defining a Skill

A skill definition captures more than executable logic. At minimum, a skill specifies:

  • intent and parameters
  • constraints and safety boundaries
  • required execution context
  • observable success criteria

This information allows the system to reason about the skill before it is executed.

Registering and Discovering Skills

Once defined, a skill is registered through RSP. Registration makes the skill discoverable by agents, comparable to other skills, and traceable across executions.

Agents query the skill registry to identify candidates that match their goals.

Execution and Feedback

When a skill is executed, runtime parameters are bound, constraints are enforced, and execution outcomes are recorded. Feedback is part of the lifecycle — enabling safer iteration, better agent decisions, and continuous improvement.
