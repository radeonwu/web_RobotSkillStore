Architecture

Layered by design,
built for interoperability.

RSP separates agent reasoning from robot control by introducing a skill interface, orchestration, and a safe runtime. This enables composition, fallback, and verifiable operations across heterogeneous robots.

Agent plans Orchestration composes Runtime executes Robot/Sim acts
Reference flow
Agent → Skill.call() → Orchestrator selects implementation → Runtime enforces guards & monitors → Robot/Sim executes actuations

Skills are selected based on a robot's capability profile and the skill's declared constraints.

Four layers

Agent layer

LLM/VLA/policy planners that decide what to do, not how to drive motors.

Orchestration layer

Skill discovery, matching, composition, retries, fallbacks, and recovery strategies.

Runtime layer

Safe execution: resource limits, safety guards, telemetry, standardized error semantics.

Robot/Simulator layer

ROS2/MoveIt/Nav2/Isaac/etc. remain underneath — integrated through skill adapters.

Protocol primitives

  • Skill.register: publish manifest + runtime endpoint + verification artifacts
  • Skill.discover: query by capability, constraints, domain, trust level
  • Skill.call: invoke with IO schema, mode (sync/async/stream), timeouts
  • Telemetry: standardized logs, metrics, traces
Interoperability
RSP is designed to coexist with existing ecosystems. You keep your ROS/Isaac stack — RSP standardizes the outer contract.
