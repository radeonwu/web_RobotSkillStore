Skill Lifecycle

PurposeThe lifecycle that makes robot skills trustworthy and reusable.

What this page coversRegister → Validate → Certify → Deploy, and how RSS governs each stage so skills become portable assets rather than project-specific code.

Lifecycle Anchor

Overview

RSS treats robot skills as capability assets with a lifecycle: Register → Validate → Certify → Deploy.

  • Register: suppliers publish skills with interfaces + constraints.
  • Validate: RSS executes skills and records runs.
  • Certify: RSS evaluates evidence and issues a certification snapshot.
  • Deploy: users reuse certified skills with predictable behavior.
