How Skill Deployment Works

An application team selects certified skills from the catalog, configures them with task-specific parameters and constraints, and orchestrates them in sequence. RSS handles the evidence and governance automatically.

Task

High-level goal (e.g. "assemble connector")

➜

Select Skills

Choose certified skills from catalog by capability & evidence level

➜

Configure

Set constraints, force limits, tolerance parameters per robot profile

➜

Evidence

Runs logged automatically — browse in Evidence Viewer

➜

Recover

On failure: adjust parameters or switch to alternate skill

➜

Execute

Run skills on real robot (or HIL/sim) — outcomes captured automatically

Example: Connector Assembly on UR5 + RG2

Workflow: pick_place → connector_insert → tighten_screw. The same three skills run on a different workcell by changing robot profile and force parameters — no code changes.

Why This Approach Scales

  • Faster iteration — failures become parameter updates with evidence, not weeks of re-coding.
  • Safer reuse — constraint declarations and maturity gates reduce deployment risk across robot variants.
  • Portable delivery — the same task plan runs on different workcells by swapping robot profiles.
  • Continuous quality — governance state updates automatically as more runs accumulate; degradation is caught before it becomes a production incident.
