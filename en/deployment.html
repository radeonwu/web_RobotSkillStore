Users & Deployment

PurposeTurn certified skills into real applications.

Lifecycle Anchor

RSS outputs skills to the user side as reusable application building blocks — not one-off demo code.

Agent-based Skill Deployment Flow

An application team typically starts with a user task (for example, to assemble connector and tighten screw using UR5 robotic arm and RG2 gripper) and lets an agent orchestrate certified skills as building blocks.

Task

Pick → insert → tighten

➜

Plan

Decompose into subtasks: pick → insert → tighten

➜

Match

Retrieve skills: pick_place, connector_insert, tighten_screw

➜

Execute

Run each skill with constraints + parameters (sim/HIL/real)

↘

Recover

On failure: retry, adjust parameters, or switch candidate skill

➜

Evidence

Attach traces (video/logs/metrics) and review in viewer

What the agent does (practical loop)

  • Pick: agent selects a grasp template + approach height for pick_place based on part pose and workspace constraints.
  • Insert: agent sets pose constraints and failure-recovery thresholds for connector_insert (approach/retract, contact, seat).
  • Tighten: agent configures tighten_screw with torque/angle targets and validates evidence for determinism.
  • Govern: each run produces a trace bundle; stability is promoted by validation → certification, not by rewriting code.
// Pseudocode: agent orchestration over certified skills
goal = "assemble_connector_and_tighten"
plan = ["pick_place", "connector_insert", "tighten_screw"]

for step in plan:
    candidates = rss.search(skill_name=step, evidence_min="E1")
    skill = agent.select_best(candidates, context)
    params = agent.configure(skill, context, constraints)
    run = rss.execute(skill, params)
    if not run.success:
        context = agent.recover(run, context)  # retry / reparam / alternate skill
        continue
    agent.update_context_from_evidence(run.evidence)

Connector assembly application example

Example workflow: pick_place → connector_insert → tighten_screw. The same skills can be reused by changing constraints and parameters, not by rewriting programs.

  • pick_place: grasp templates + collision-aware IK.
  • connector_insert: pose constraints + approach/retract + recovery traces.
  • tighten_screw: deterministic tightening with traceable torque/angle evidence.

Why agent orchestration matters

Agent orchestration is not just a recommendation — it is how application teams scale delivery. The agent automates the integration work (plan/match/execute/recover) while RSS keeps each skill modular, evidence-backed, and governable.

  • Faster iteration: failures become parameter updates with evidence, not weeks of re-coding.
  • Safer reuse: constraints and maturity gates reduce deployment risk across variants.
  • Portable delivery: the same plan can run on different workcells by swapping skill bindings.
