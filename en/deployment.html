How Skill Deployment Works

An application team selects certified skills from the catalog, configures them with task-specific parameters and constraints, and orchestrates them in sequence. RSS handles the evidence and governance automatically.

Task

High-level goal (e.g. "assemble connector")

➜

Select Skills

Choose certified skills from catalog by capability & evidence level

➜

Configure

Set constraints, force limits, tolerance parameters per robot profile

➜

Evidence

Runs logged automatically — browse in Evidence Viewer

➜

Recover

On failure: adjust parameters or switch to alternate skill

➜

Execute

Run skills on real robot (or HIL/sim) — outcomes captured automatically

Example: Connector Assembly on UR5 + RG2

A complete deployment walkthrough — from task goal to evidence record — using the flow above.

1
Task
Goal: Assemble harness connectors on UR5 workcell #2
3-step sequence — locate connector, insert with compliance, torque-fasten
2
Select Skills
pick_place v1.2.0 — grips connector from tray using 2.5D vision guidance CERTIFIED
connector_insert v2.1.0 — compliant force-controlled insertion with depth feedback CERTIFIED
tighten_screw v1.0.3 — torque-controlled fastening with turn-count confirmation CERTIFIED
3
Configure
robot_profile:  ur5_rg2_workcell2
gripper:        OnRobot RG2
force_limit:    18 N          # connector_insert constraint
torque_target:  0.45 N·m     # tighten_screw target
position_tol:   ±0.4 mm
vision_mode:    2.5D          # pick_place depth-assisted grasp
on_failure:     retry(2) → alert

Parameters bind the certified skill to this specific robot and workcell — the skill code itself is unchanged.

4
Execute

Skills run in sequence: pick_place → connector_insert → tighten_screw. Each skill executes within its declared constraint envelope; outcomes are captured per-run automatically.

↺
Recover (if needed)
If connector_insert exceeds force limit: RSS flags the run, retries with a relaxed force_limit, or proposes switching to an alternate certified skill (e.g. connector_insert_compliant_v2). No code change — parameter adjustment only.
5
Evidence
[08:32:11]  pick_place@1.2.0        PASS  force=12.1 N    grasp_conf=0.94
[08:32:18]  connector_insert@2.1.0  PASS  peak_force=16.8 N  depth_err=0.2 mm
[08:32:27]  tighten_screw@1.0.3     PASS  torque=0.44 N·m   turns=3

3/3 skills passed · run archived · governance snapshot updated

All run records are stored automatically and accessible from the Evidence Viewer. Governance state updates as runs accumulate — the same skill reused on a second workcell adds to its evidence base without any extra steps.

Why This Approach Scales

  • Faster iteration — failures become parameter updates with evidence, not weeks of re-coding.
  • Safer reuse — constraint declarations and maturity gates reduce deployment risk across robot variants.
  • Portable delivery — the same task plan runs on different workcells by swapping robot profiles.
  • Continuous quality — governance state updates automatically as more runs accumulate; degradation is caught before it becomes a production incident.
